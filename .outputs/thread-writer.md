thread: enterprise agents fail at scale

1/ 88% of enterprise AI agent pilots never reach production. The 12% that survive report 171% ROI. Everyone's citing the ROI. No one's asking why 88% failed.

2/ The gap isn't model quality — it's infrastructure. Actual TCO for enterprise agents runs 3.4x API-only estimates. You budget for the API. You don't budget for orchestration, evals, human review loops, and security. Those are where agents die.

3/ Security is the quietest kill. 97M monthly MCP downloads. 10,000 servers deployed. 66% have known security flaws. Enterprises with legal and compliance review will not ship tools with that track record. Most don't know the flaws are there yet.

4/ Eight percent of companies running agent pilots have an incident response plan for agent failures. 8%. The rest are one prompt injection away from a public incident — and building toward production anyway.

5/ The 171% ROI figure isn't wrong. It's survivorship. You're measuring projects that cleared legal, survived prod incidents, and had a champion for 18 months. That's not "enterprises using agents." That's the luckiest 12% of enterprises using agents.

6/ The failure modes cluster. Not hallucination — that's solvable. It's: no eval harness, no rollback path, no explainability for regulated industries, no budget for operational overhead. Infrastructure built assuming the model works. No plan for when it doesn't.

7/ The builders who ship aren't smarter. They built for the 88% case, not the demo. Eval infrastructure before prod. Security review before scale. Operational budget for overhead, not just tokens. The unlock for enterprise AI isn't better models — it's better plumbing.
